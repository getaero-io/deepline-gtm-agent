# Deepline Workflow Spec Skill

## Workflow

1. Define the business job.
2. Define inputs and output contract.
3. Identify Deepline provider/tool path.
4. Add validation and fallback rules.
5. Specify one-row pilot and full-run gate.
6. Specify Deepline Session UI plan, logs, usage, and output lineage expectations.

## Output Pattern

- workflow name
- trigger
- inputs
- steps
- provider/tool assumptions
- fallback path
- validation rules
- output schema
- pilot command
- approval gates
- observability checklist
