# Deepline GTM Context For Hermes

Keep this context small. This is the durable frame the Hermes GTM agents should start from.

## Company Frame

Deepline is the GTM data and execution layer for teams building GTM workflows with AI agents and coding agents.

Best current homepage expression:

> Build GTM workflows in Claude Code. Deepline makes the data, tools, and execution reliable.

Alternate short expressions:

- Go-to-market is code now.
- Agents are the interface. Deepline is the GTM data and execution layer.
- Deepline helps GTM engineers build revenue systems with agent-ready data, enrichment waterfalls, provider routing, validation, and workflow APIs.

## What Deepline Should Own

The durable layer is not the agent. Agents will change. Deepline should own:

- provider routing
- enrichment waterfalls
- data validation
- GTM-specific context
- workflow execution rails
- run logs and observability
- Session UI plans and progress
- usage, credit, and limit visibility
- output lineage
- bring-your-own-key support
- workspace data store
- business definitions
- workflow state and repeatability

## Primary ICP

Primary:

- GTM engineers
- technical RevOps teams
- agency operators building repeatable outbound/research/enrichment systems
- founders or GTM leaders already trying Claude Code/Codex/Hermes for revenue workflows

Secondary:

- CROs and founders who need the business translation: their team wants to use agents for GTM, and Deepline makes those agents reliable enough to run real workflows.

## Common Alternatives

- Clay tables
- one-off Claude Code scripts
- direct provider APIs
- RevOps manual work
- data engineering tickets
- brittle Zapier/n8n automations

## Core Buyer Pain

The pain is not "no data." The pain is:

- choosing the right provider
- falling back when the provider misses
- validating that data is correct
- making workflows repeatable
- keeping costs controlled
- landing output where the team already works
- turning company-specific GTM context into execution logic

## Messaging Rules

Say:

- "If you know the one API you need, call it directly. If you need the right data across many providers, Deepline gives your agent the tools, routes, and validation logic."
- "Clay is a table for GTM workflows. Deepline is the API and execution layer for GTM agents."
- "Send results where your team works. Monitor the workflow in Deepline."
- "Turn wins, losses, and ICP notes into reusable GTM logic."

Avoid:

- "AI-powered GTM"
- "automation platform"
- "data platform"
- "semantic layer"
- "orchestration layer"
- "duct-tape"
- "unlock"
- "transform"
- broad unsupported claims
