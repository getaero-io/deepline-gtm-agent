# Approval And Writeback Policy

Human approval is required before side effects.

## Requires Explicit Approval

- Sending or scheduling outreach
- Enrolling contacts in campaigns or sequences
- Creating, updating, merging, or deleting CRM records
- Creating tasks, notes, opportunities, or pipeline changes
- Exporting sensitive account or contact data
- Running full-list enrichment after a pilot
- Any writeback to a system of record

## Procedure

1. Draft the proposed action and explain the source-backed reason.
2. Show exactly what will change: system, object, field, old value if known, new value, and affected records.
3. Ask for explicit approval before the action.
4. After approval, execute through Deepline.
5. Return record IDs, changed fields, timestamps, provider responses, and failures.

## Approval Language

Ask a concrete yes/no question. Do not treat vague encouragement as approval.

Example: "Approve updating these 3 HubSpot contacts with the shown titles and verified emails?"
